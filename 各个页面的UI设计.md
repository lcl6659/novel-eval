# 网文开篇测评系统 — UI 设计与交互规范

## 全局设计规范

### 设计风格

- **整体基调**：专业但不冰冷，文学感 + 科技感结合。类似"豆瓣读书"的文艺气质 + "产品分析工具"的数据专业感
- **主色调**：深靛蓝 `#1a1a2e` + 暖金色 `#d4a853`（强调/CTA）+ 纯白 `#ffffff`（背景）
- **辅助色**：浅灰蓝 `#f0f2f5`（区块背景）、成功绿 `#52c41a`、警告橙 `#fa8c16`、信息蓝 `#1677ff`
- **字体**：系统字体栈，标题使用较粗字重营造"书卷气"
- **圆角**：卡片 12px，按钮 8px，输入框 8px
- **阴影**：卡片使用柔和阴影 `0 2px 8px rgba(0,0,0,0.06)`

### 响应式断点

| 断点 | 宽度 | 布局策略 |
|------|------|---------|
| 桌面 | ≥1200px | 最大内容宽度 1200px，居中 |
| 平板 | 768px~1199px | 自适应，侧边栏收起 |
| 手机 | <768px | 单列布局，底部导航 |

### 全局导航栏 (Header)

**桌面端**：

**组件与交互**：
- **作品标题**：Ant Design `Input`，必填，最大 50 字
- **作品类型**：Ant Design `Select` 或 `Radio.Group` 以标签形式展示，单选，选项包含全部 10 个类型（玄幻/都市/仙侠/科幻/言情/悬疑/历史/游戏/轻小说/现实），必填
- **开篇内容**：Ant Design `Input.TextArea`，最小高度 400px，支持自动拓展高度
  - 实时字数统计显示在右下角
  - 字数 <5000：红色提示 "至少需要 5,000 字"
  - 字数 5000~10000：绿色 "✓ 字数合格"
  - 字数 >10000：橙色警告 "超出上限，仅前 10,000 字将被评测"
- **提示文字**：使用 Ant Design `Alert` type="info"，放在文本框下方
- **下一步按钮**：字数不达标时禁用(disabled)，达标后高亮

## 交互动效总结

| 场景 | 动效 | 实现建议 |
|------|------|---------|
| 首页 Hero 区 | 背景微光粒子 / 渐变流动 | CSS animation 或轻量 Canvas |
| 首页数字统计 | CountUp 数字跳动 | react-countup 库 |
| 首页工作流 | 滚动到视口时淡入上浮 | Intersection Observer + CSS transition |
| 读者卡片横滑 | 自动缓慢滚动 + 手动滑动 | CSS scroll-snap 或 Swiper |
| 提交表单步骤 | 切换时左右滑动 | Ant Design Steps + CSS transition |
| 评测进度数字 | 实时跳动更新 | SSE 推送 + 数字递增动画 |
| 读者动态消息 | 从下方滑入 | CSS @keyframes slideUp |
| 评测完成 | 撒花动画(confetti) | canvas-confetti 库 |
| 报告雷达图 | 加载时从中心展开 | ECharts 内置动画 |
| 卡片 hover | 上浮 + 阴影加深 | CSS transform + box-shadow transition |
| 按钮 hover | 微放大 + 颜色加深 | CSS transform: scale(1.02) |
