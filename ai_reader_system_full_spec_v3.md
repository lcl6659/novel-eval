# 一、最大问题：读者人格还不够“稳定”

你现在的人设是 **prompt-based persona**。

这在 10 个 agent 时很好用，但到 **1000 个**会出现一个问题：

**LLM 的“基础人格”会覆盖人设人格。**

典型表现：

即使人设不同，评价仍然类似：

```
节奏有点慢
设定不错
人物还可以
建议增加冲突
```

因为模型在走 **默认审稿模式**。

---

## 解决方案：增加“认知偏差层”

给每个读者增加 **判断偏置参数**。

例如：

```json
{
  "taste_bias": {
    "prefer_fast_pacing": 0.8,
    "care_about_writing": 0.3,
    "love_power_fantasy": 0.9,
    "romance_sensitivity": 0.1
  }
}
```

在 prompt 里明确：

```
当节奏慢时你会非常不耐烦
当设定普通时你会直接降低评分
当主角不够强时你会失去兴趣
```

这样模型会产生 **真正不同的评分分布**。

否则：

**1000 个读者的评分曲线会高度重合。**

---

# 二、评分体系需要防止“AI评分病”

LLM 有一个典型问题：

**评分过于集中。**

常见结果：

```
6.8
7.1
6.5
7.0
6.9
```

所有人都在 **6~8分之间**。

真实读者不会这样。

---

## 建议增加：评分风格

每个读者增加：

```json
"rating_style": "harsh | generous | polarized"
```

规则：

### 严格型

```
好书：8
普通：5
差：3
```

### 宽容型

```
好书：9
普通：7
差：5
```

### 极端型

```
喜欢：9
不喜欢：3
```

这样：

**评分分布才像真实平台。**

# 三、增加“阅读行为模拟”（很关键）

这比评分 **价值高很多**。

# AI 读者评测系统（AI Reader Evaluation System）

Version: **V2.0**
目标：构建一个由 **大量 AI Reader Agent** 组成的小说评测系统，模拟真实读者阅读行为，对网文开篇进行 **评价、市场预测、修改建议**。

---

# 一、产品目标

帮助作者回答三个核心问题：

1️⃣ **读者会不会喜欢这本书**
2️⃣ **问题出在哪**
3️⃣ **应该怎么改**

系统模拟 **100~1000名AI读者**，生成：

* 阅读行为
* 评分
* 评论
* 共识分析
* 市场预测
* 修改建议

最终输出 **一份完整评测报告**。
